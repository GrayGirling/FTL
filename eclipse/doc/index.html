Using Eclipse to Develop FTL Code

Eclipse is a powerful Integrated Development Environment that you can use to write, develop and debug C applications (among other things) on your Windows, Mac or Linux computer.

Find out (much) more about it here and download the 'lunar' release here.

There are two ways in which Eclipse can be used:

  1. To develop the FTL library
  2. To develop a new FTL-using utility

Before you can build libraries or utilities in C or C++ you will need to ensure that your operating system has the appropriate tools installed. This process is described in the documentation at FTL-build-setup.

Eclipse Workspace Location

The Eclipse projects provided are designed to be placed in a subdirectory held in the same directory as the Git source. (That is both the Eclipse workspace subdirectory and the git subdirectory should be in the same directory.)

For example the following commands would create two directories 'FTL' and 'ws' for the git sources and the Eclipse workspace respectively.

git clone https://github.com/GrayGirling/FTL.git

mkdir ws

Once created you should select the sibling directory ('ws' above) as the workspace Eclipse normally asks for when it starts. (If your Eclipse does not use File > Change Workspace... to select it after it starts.)

Developing a new FTL-using utility

Two Eclipse projects are available to import from the 'ecliipse' subdirectory of the FTL git repository. They are

  • ftl_bin_dev - this project contains pre-compiled versions of the FTL library and their associated headers
  • ftl_cmd - this project contains a simple example project for creating the FTL-using utility called 'ftl' - all it does is supports ftl commands. You can use it as a proforma for developing your own utility.

If you have a new workspace it will contain no projects. To import these projects in to the workspace use File > Import ... then select the type of import you want to make under General > Existing Projects into Workspace (Next>). Use the 'select archive file' radio button and browse to FTL/eclipse where you can select the relevant project.

Do this for both the above projects.

After archive ftl_bin_dev.zip is imported the following projects will appear:

  • release_Debug - containing the binaries and headers to use when building utilities for debug; and
  • release_Release - containing the non-debug binraries and headers.

On importing the ftl_cmd-project.zip file this project will be created

  • ftl_cmd - containing the source of a simple utility 'ftl'.

There are many versions of the utility that can be built depending on which operating system you use. To select the one you want to build select the ftl_lib project and right-click selecting Build Configurations > Set Active. The name of each build configuration is made up of three parts:

  1. Release_ or Debug_ depending on whether you need a debuggable version of the utility
  2. Linux, Mac or MinGW depending on which operating system you are using (use MinGW for Windows)
  3. Optionally '_norl' for versions that do not use the 'readline' library (and are therefore not subject to its licensing conditions)

For example a version for debugging on Linux with 'readline' support will be built with the build configuration 'Debug_Linux'.

Once you have set the active project you can clean and/or build it by selecting the relevant entry when right-clicking on the project.

The utility created by the build will be copied in to your workspace directory 'release_<Release|Debug>/<operating_system>_<architecture>/bin'.

The code files in this project can be found in your workspace. They are a copy of those in the git repository, and updating them has no effect on the versions held by git. You may safely, for example, rename 'ftl.c' to the name of your own utility. You can also rename the project.

Developing the FTL Library

Two Eclipse projects are available to import from the 'eclipse' subdirectory of the FTL git repository. They are

  • ftl_bin_dev - this project contains pre-compiled versions of the FTL library and their associated headers.
  • ftl_lib-project - this project contains the FTL library source. When it is built it copies new content into ftl_bin_dev.

If you have a new workspace it will contain no projects. To import these projects in to the workspace use File > Import ... then select the type of import you want to make under General > Existing Projects into Workspace (Next>). Use the 'select archive file' radio button and browse to FTL/eclipse where you can select the relevant project.

Do this for both the above projects.

After archive ftl_bin_dev.zip is imported the following projects will appear:

  • release_Debug - containing the binaries and headers to use when building utilities for debug; and
  • release_Release - containing the non-debug binraries and headers.

On importing archive ftl_lib-project.zip this project will appear:

ftl_lib - containing the source of the FTL library.

When ftl_lib is built binaries are copied into the relevant "release_' project (where it can be picked up by FTL-using utilities)

As well as debug and non-debug releases the library can support the open source 'readline' library. When it does support this library and commands are typed to a FTL-using utility it is possible to access a line history and perform a fair amount of line editing. These features are not necessarily available when 'readline' is not provided.

Your system must be able to provide 'readline' in order to build the readline versions of the libraries (e.g. Windows does not). Also, because readline is available through a GNU Public License (GPL) there are some restrictions about what you can do with the utility you build. (Essentially you may have to make its source publically available.)

There are four versions of the library that can be built:

  • a debug version without any 'readline' support (libftln)
  • a debug version with 'readline' support (libftl)
  • a non-debug version without any "readline' support (libftln)
  • a non-debug version with 'readline' support (libftl)

The one that is to be built can be selected by a debug configuration (select the ftl_lib project and right-click selecting Build Configurations > Set Active).

Before you can use Eclipse to build any of the above you will need to set the toolset appropriately. You can do this by selecting the ftl_lib project and right-click selecting Properties. Expand the C/C++ Build section and select Tool Chain Editor. In that dialogue under 'Current Toolchain' select the

following from the dropdown list:

  • on Windows select 'MinGW GCC'
  • on Linux select 'Linux GCC'
  • on Mac OSX select 'MacOSX GCC'

Once you have set the active project you can clean and/or build it by selecting the relevant entry when right-clicking on the project.

The library created by the build will be copied in to your workspace directory 'release_<Release|Debug>/<operating_system>_<architecture>/lib'.

If you have an Eclipse project to build your own utility project set up as above it will use the new library when it is cleaned and rebuilt.
